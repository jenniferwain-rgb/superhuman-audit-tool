‚úÖ Self-Audit Checklist

Use this checklist before finalizing any UI copy

1. Crisp (Clarity-Focused)

  • Would I say this out loud to a friend?
  • Is this the shortest way to say it?
  • No corporate jargon?
  • Active voice?

2. Uplifting (Warm & Empowering)

  • Does this make someone feel more capable?
  • Emphasizes human agency?
  • Positive framing?
  • Speaks to people, not "users"?

3. Grounded (Confident Without Hype)

  • Enthusiasm feels earned, not inflated?
  • No vague superlatives?
  • Specific and tangible?
  • Can I back up this claim?

4. Strategically Witty (When Appropriate)

  • Does wit serve clarity (or compete with it)?
  • Is it evergreen?
  • Work-appropriate?
  • If no wit, is that okay?

5. Tone Match

  • In-product ‚Üí Crisp + Uplifting?
  • Marketing ‚Üí Grounded + Witty + Uplifting?
  • Sales ‚Üí Crisp + Grounded + Uplifting?

6. Style Compliance

  • Sentence case for UI copy?
  • No periods on buttons/CTAs?
  • Consistent terminology?
  • No exclamation marks unless celebrating?

UI Copy Audit Methodology

Systematic framework to evaluate existing UI copy against Superhuman voice & tone strategy

Part 1: Copy Extraction Method

Step 1: Identify copy-bearing layers

  • TEXT nodes (labels, body copy, headings)
  • COMPONENT instances with text properties
  • Button labels, form fields, tooltips
  • Empty states, error messages, success messages
  • Onboarding flows, modal dialogs

Step 2: Categorize by UI pattern

Categories: - CTAs (Call-to-Action buttons) - Headlines/Subheads - Body copy - Microcopy (labels, hints, tooltips) - Error messages - Success messages - Empty states - Onboarding/Welcome flows

Step 3: Capture metadata

For each string, record:

  • String ID
  • Figma node ID and URL
  • Coda doc reference
  • Category and surface area
  • User context
  • Current copy text
  • Character count

üìö Reusable Content Library

Pre-approved copy snippets from the Superhuman language guidance. Copy and adapt these examples for your UI.

How to use: These examples follow our voice & tone guidelines. Use them as templates or inspiration for consistent messaging across the product.

Error Messages

Format: [Error nature] + [How to resolve]

Tone: Crisp and clear only, no wit

Examples

Generic Error
"Data not found. Try something else."
Content Limit
"Too much text to review. Proofreader can help with text up to 10,000 words."
Agent-Specific
"[Agent name] encountered an issue. [Specific next step]."
Save Failed
"Couldn't save your changes. Try again?"

Success States

Format: [Encouraging statement] + [What was achieved]

Tone: Varies by persona

For Students (60% Crisp, 40% Uplifting)

"Nicely written."
"Well crafted."
"You're all clear."
"Nicely done! You have no Proofreader suggestions, but you can try another agent for more customized feedback."

For Knowledge Workers (60% Uplifting, 40% Crisp)

"Your writing is looking polished and tidy."
"Well done! Your writing is on point."
"Spot on! Your writing really hit the mark."
"That's a well-worded piece of prose!"

For General/Playful Agents (Even Split)

"Looking good! Keep up the great work."
"Nice! You're ready to go."

Empty States

Format: [Uplifting readiness] + [What's needed to start]

Tone: Uplifting and informative, avoid negative framing

Examples

Playful with Product Name
"Ready to Go whenever you are! Add text to get started."
Direct and Clear
"Write at least 50 words to get started."
Agent-Specific
"[Agent] is ready to help once you have [X words/content]."
Encouraging
"Start writing and I'll help you along the way."

‚ùå Avoid These Patterns

"You haven't written enough." ‚Üí Negative framing
"Not enough content to analyze." ‚Üí Blaming tone
"Nothing to see here yet!" ‚Üí Dismissive

Body Copy (Primary Agent Copy)

Format: [Action taken] + [Next steps/recommendations]

Tone: 80% Crisp, 20% Uplifting

Examples

With First-Person
"I found 11 claims without citations in your text."
Analysis Result
"I flagged phrases that often appear in AI writing."
With Recommendation
"I reviewed your text for clarity. Consider breaking up long sentences for better readability."
With Hedging
"I detected [N] potential issues. Review each to see if changes are needed."

Chat/LLM Responses

Tone: Conversational, always use first-person ("I")

Emphasis: Assisting and empowering, not replacing the user

Examples

Greeting
"Hi, Jane. I'm here to help you craft your best writing."
Explaining Capability
"I'm an AI, so I don't have likes or dislikes. However, I can help you analyze Shakespeare's writing style."
In Progress
"Reviewing options to improve your text."
Clarification
"I can help with that. Could you provide more details about [specific aspect]?"
